## Github Actions Overview

The basic overview of github actions is when you enter a change into your repository you can create a pipeline to execute specific tasks. You create the workflows so that when a specific event happens in your repo a specific action is triggered in the workflow to execute in response to the action that happened. You can create a workflow to be triggered by pushes, pull requests, merges, and creation or deletion. For example when code is pushed run the tests or when code is merged redepoly the app.

Most common uses of github actions

- Continuous Integration
- Continuous Deployment
- Code Quality and Linting
- Automation Tasks
- Scheduled Jobs

## My Experiment

Because we have already created a workflow in this class and we have already used some of these application examples, I decided to use a different application that we haven't used up until this point. I set it up to do a scheduled workflow. I created a workflow that automatically redeploys the application every Monday at 9:00 a.m. using a cron schedule. I chose this scenario because in real-world development, teams often avoid deploying changes late in the week (such as on Fridays) to prevent having to fix issues over the weekend.

By scheduling deployments for Monday mornings, developers can:

Ensure deployments happen at a controlled time
Avoid risky late-week releases
Start the week with a fresh and updated application

This experiment demonstrates that GitHub Actions is not only useful for reacting to code changes, but can also be used for time-based automation, similar to a server running scheduled jobs.

---

### Github Action Code Example

```yaml
name: Weekly Deployment

on:
  schedule:
    - cron: "0 9 * * 1" # Every Monday at 9:00 AM (UTC)

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v3

      - name: Install dependencies
        run: npm install

      - name: Build project
        run: npm run build

      - name: Deploy application
        run: echo "Deploying application..."
```
